# Game Definition: Home Game

## System Identifier

- `system_id`: `dnd5e`
- `display_name`: `Dungeons & Dragons 5th Edition`
- `game_id`: `home-game`
- `game_name`: `Home Game`

## Purpose

This file defines the assumptions the planner should use when generating session plans for the named game `home-game`, which uses D&D 5e.

## Core Rules Guardrail

When generating output for this folder:

- use D&D 5e terminology
- use D&D 5e style stat blocks or base-game-equivalent creature references
- keep rewards and treasure close to core 5e expectations unless the GM explicitly asks otherwise
- do not use Pathfinder 2e mechanics, math, monster conventions, or treasure assumptions

## Terminology

Prefer terms like:

- armor class
- hit points
- saving throw
- skill check
- challenge rating
- spell slot
- long rest
- short rest

Avoid Pathfinder 2e-specific terms unless the user explicitly asks for conversion.

## Threat And Stat Expectations

- combat-ready enemies should have 5e-usable stat blocks or direct base-game references
- threats may include traps, hazards, social pressure, and environmental danger in addition to monsters

## Reward Expectations

- rewards should feel plausible for core 5e play
- prefer modest treasure, consumables, clues, favors, and magic items appropriate to level and tone
- avoid importing Pathfinder-style treasure balance assumptions

## Output Style

- practical prep first
- evocative but concise
- fantasy-forward by default unless the session tone says otherwise
